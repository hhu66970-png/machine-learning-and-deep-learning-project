# 模型微调技术探索（PEFT）—— 实验方案与原理讲解

> 课题对应 PPT「选题④ 模型微调技术探索」。参考文献：LoRA、BitFit、AdaptFormer、SSF。
> 设备：RTX 5070 Ti Laptop（12GB）。环境：conda `dl_course`（PyTorch 2.10 + CUDA 12.8 + timm 1.0.25）。

---

## 一、一句话说清楚我们在干什么

我们拿一个**已经在超大数据集（ImageNet-21k，1400 万张图、2 万类）上训练好的视觉模型（ViT-Base）**，
让它去学一个**新的小数据集**（比如 102 类花、37 类宠物）。

直接重训整个大模型既贵又容易过拟合。于是我们研究一类技术：
**冻住大模型 99% 的参数，只训练极少量（常常 < 1%）的"小补丁"参数**，
就能达到接近、甚至超过"全部重训"的效果。这类技术叫
**参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）**。

本课题的核心工作 = **横向对比 7 种微调策略**，在多个数据集上比较它们的
「精度 / 可训练参数量 / 显存 / 训练时间」，并分析背后的规律，最后尝试改进。

## 二、为什么要这么做（原理的来龙去脉）

### 2.1 迁移学习：站在巨人的肩膀上
一个在 ImageNet-21k 上训练好的 ViT，它的中间层已经学会了通用的视觉特征
（边缘、纹理、形状、物体部件……）。这些特征对几乎所有图像任务都有用。
所以面对一个新的小任务，我们**不需要从零学起**，只要"微调"一下让它适配新任务即可。
这就是迁移学习（Transfer Learning）。

### 2.2 两个朴素做法，各有毛病
- **全量微调（Full Fine-Tuning）**：把整个模型的参数都拿来继续训练。
  - 优点：上限高，效果通常最好。
  - 缺点：① 要更新 86M 个参数，**显存/算力开销大**；② 每个新任务都得**存一份完整的大模型**（86M×4字节≈330MB），10 个任务就 3.3GB；③ 小数据上**容易过拟合**、甚至把预训练学到的好特征"训坏"（灾难性遗忘）。
- **线性探测（Linear Probing）**：冻住整个骨干网络，只在最后接一个新的线性分类头来训练。
  - 优点：极省、极快、绝不破坏预训练特征。
  - 缺点：表达能力太弱，**精度通常明显低于全量微调**（因为骨干一点都没适配新任务）。

> 这两者构成了我们实验的**下限（Linear Probing）和上限（Full FT）两条基准线**。
> PEFT 方法的目标就是：**用接近 Linear Probing 的代价，逼近甚至超过 Full FT 的精度。**

### 2.3 PEFT 的核心思想
> "**冻结预训练大模型，只插入/挑选极少量可训练参数**。"

不同 PEFT 方法的区别，仅仅在于"**那一小撮可训练参数长什么样、插在哪**"。
理解了这句话，下面 5 种方法你就都懂了。

## 三、七种方法逐个讲（带直觉）

先回忆 ViT 的一个 Transformer Block 的结构（这是"插补丁"的位置）：
```
输入 x
 └─► LayerNorm ─► Multi-Head Self-Attention(Q,K,V 投影) ─►(+残差)──┐
                                                                   ▼
 └─────────────► LayerNorm ─► MLP(Linear→GELU→Linear) ─►(+残差)──► 输出
```

| # | 方法 | 训练什么 | 一句话直觉 |
|---|---|---|---|
| 0a | **Linear Probing** | 仅分类头 | 下限基准：骨干完全冻结 |
| 0b | **Full Fine-Tuning** | 全部参数 | 上限基准：全部重训 |
| 1 | **BitFit** | 只训练所有 bias + 头 | 连权重矩阵都不动，只微调每层的"偏置旋钮" |
| 2 | **LoRA** | 给注意力的 Q/V 投影加低秩旁路 | 大矩阵的更新量其实是"低秩"的，用两个小矩阵 B·A 表示 |
| 3 | **Adapter / AdaptFormer** | 在 MLP 旁并联一个小瓶颈网络 | 给每个 block 插一个"小转接头"做特征调整 |
| 4 | **SSF** | 给每层特征加可学习的缩放γ和平移β | 只对特征做"拉伸 + 平移"两步线性微调 |
| 5 | **VPT** | 在输入序列前拼接可学习的提示 token | 给模型喂几个"暗号 token"引导它适配新任务 |

### 1. BitFit（只训 bias）
神经网络每一层是 `y = Wx + b`。BitFit **冻结所有权重 W，只训练偏置 b**（外加分类头）。
可训练参数 ≈ 0.1%。直觉：bias 像每层的"微调旋钮"，只拧旋钮就能让模型偏向新任务。
> 论文：BitFit (ACL 2022)。最简单、参数最少，是很好的"极简对照组"。

### 2. LoRA（低秩适配）⭐
对一个权重矩阵 W（d×k），全量微调要学一个同样大的更新量 ΔW。
LoRA 的洞见是：**这个 ΔW 往往是"低秩"的**，可以用两个瘦长矩阵的乘积近似：
```
W_new = W(冻结) + ΔW，  ΔW = B·A
其中 B 是 d×r，A 是 r×k，秩 r ≪ d（比如 r=4 或 8）
只训练 A、B，参数量从 d×k 降到 r×(d+k)
```
通常只加在注意力的 Q、V 投影上。推理时可把 B·A 合并回 W，**零额外延迟**。
> 论文：LoRA (ICLR 2022)，原为大语言模型设计，现已是 PEFT 的事实标准。
> 关键超参：秩 `r`、缩放 `alpha`、加在哪些层。这些都是我们可做消融的点。

### 3. Adapter / AdaptFormer（瓶颈转接头）
在每个 Transformer block 的 MLP 旁边**并联**一个小网络：
```
down(降维 d→r) → GELU → up(升维 r→d)，再乘一个缩放系数 s
```
冻结原模型，只训练这些 adapter（+头）。AdaptFormer 是专门为 ViT 设计的并联版 adapter。
直觉：给每层加一个"小补习班"，专门学新任务需要的额外变换。
> 论文：AdaptFormer (NeurIPS 2022)。

### 4. SSF（Scale & Shift Features）⭐（PPT 主打文献）
在网络的若干操作之后，对特征做一次**逐通道的缩放和平移**：
```
y = γ ⊙ x + β     （γ、β 是可学习向量，⊙ 是逐元素相乘）
```
只训练所有的 γ、β（+头），参数极少。最妙的是：γ/β 是线性变换，
**推理时可以被"吸收"进前一层的权重里（重参数化），实现零额外开销**。
直觉：预训练特征已经很好了，只需对每个通道做"调亮度/调对比度"式的微调。
> 论文：SSF (NeurIPS 2022)。这是 PPT 指定的代表文献，建议作为重点方法。

### 5. VPT（Visual Prompt Tuning，可选加分项）
不改模型内部，而是在输入的 patch token 序列**前面拼接 N 个可学习的"提示 token"**，
冻结骨干只训练这些 prompt（+头）。直觉：用几个"暗号"引导冻结的模型走向新任务。
> 经典基准方法（ECCV 2022），不在指定文献里，但加上能让对比更全面 → 加分。

## 四、我们测什么、怎么算"分析详尽"

每个方法 × 每个数据集，都记录这几列，汇成大表格（这是报告内容的主体）：

| 指标 | 含义 | 为什么重要 |
|---|---|---|
| **Top-1 Accuracy** | 测试集准确率 | 效果好不好 |
| **可训练参数量 / 占比** | 训了多少参数（个 & %）| PEFT 的核心卖点 |
| **峰值显存** | 训练时显存占用 | 工程可行性 |
| **训练时间 / 吞吐** | 每 epoch 耗时 | 效率 |
| **存储成本** | 每个任务需额外保存多少参数 | 多任务部署成本 |

> **"详尽分析"= 不止给一张表，还要画图 + 解释规律**，例如：
> - 「精度 vs 可训练参数量」散点图（看谁在"少参数高精度"的甜区）
> - LoRA 秩 r ∈ {1,2,4,8,16} 的消融曲线
> - 不同数据集上方法排名是否变化（细粒度 vs 通用任务）
> - 收敛速度对比（PEFT 是否比全量微调收敛更快/更稳）

## 五、实验矩阵（初步设计，可调）

**骨干模型**：`vit_base_patch16_224.augreg_in21k`（ImageNet-21k 预训练，timm 提供）

**下游数据集**（torchvision 可自动下载，覆盖通用+细粒度+纹理）：
| 数据集 | 类别数 | 类型 | 规模 |
|---|---|---|---|
| CIFAR-100 | 100 | 通用物体 | 50k/10k |
| Oxford Flowers-102 | 102 | 细粒度(花) | 小 |
| Oxford-IIIT Pet | 37 | 细粒度(宠物) | 中 |
| DTD | 47 | 纹理 | 小 |
| (可选) FGVC-Aircraft | 100 | 细粒度(飞机) | 中 |

**方法**：Full FT、Linear Probe、BitFit、LoRA、AdaptFormer、SSF、(可选)VPT —— 共 6~7 种。

**消融实验**（拉满内容）：LoRA 的秩 r、AdaptFormer 的瓶颈维度、SSF 插入位置、数据量敏感性（10%/50%/100% 训练数据）。

**改进方向（冲 90+）**：组合策略（如 LoRA+SSF）、自适应秩、只在部分层插补丁等，提出你自己的小改进并验证。
