# 1. 引言

## 1.1 预训练-微调范式与视觉Transformer的崛起

近十年来,深度学习领域逐步形成了一种主导性的工作流程:**先在超大规模数据上预训练一个通用模型,再针对具体下游任务进行微调**。这一"预训练-微调"(pre-training then fine-tuning)范式之所以成功,根源在于大规模预训练能够让模型习得高度可迁移的通用表征——从低层的边缘、纹理、颜色,到中层的形状、物体部件,再到高层的语义概念。面对一个新的、往往规模有限的下游任务时,模型不必从随机初始化"白手起家",而是站在已有表征的肩膀上,只需将通用知识"适配"到具体任务即可。这极大降低了对标注数据量的需求,也显著提升了收敛速度与泛化性能。

视觉Transformer(Vision Transformer, ViT)的提出将这一范式推向了新的高度[1]。ViT摒弃了卷积网络长期依赖的局部归纳偏置,转而将图像切分为若干固定大小的图块(patch),把每个图块线性投影为一个token,再叠加位置编码后输入标准的Transformer编码器,通过多头自注意力机制建模图块之间的全局依赖。这种设计虽然在小数据上不如卷积网络高效,但在海量数据上展现出更强的可扩展性:当在ImageNet-21k(约1400万张图像、2万余类别)乃至更大数据集上预训练时[14],ViT的表征质量与下游迁移能力全面超越同等算力的卷积网络。本文所采用的骨干正是timm库提供的、在ImageNet-21k上预训练的ViT系列模型[13]。随着模型规模从ViT-Small(约22M参数)增长到ViT-Base(约86M)再到ViT-Large(约304M),其表征能力持续增强,但同时也对微调环节的算力与存储提出了更严峻的挑战。

## 1.2 全量微调的三大痛点

最直接的微调方式是**全量微调(Full Fine-Tuning)**:解冻预训练模型的全部参数,连同新增的分类头一起在下游数据上端到端地继续训练。全量微调通常能达到很高的精度上限,长期以来被视为迁移学习的默认做法。然而,当骨干规模膨胀到数千万乃至数亿参数时,全量微调暴露出三个日益严重的痛点。

**第一,算力与显存开销大。** 全量微调需要对模型的每一个参数计算梯度并维护优化器状态(以AdamW为例,每个参数还需额外保存一阶与二阶动量[12]),反向传播必须贯穿整个网络。以ViT-Base为例,需更新的可训练参数高达85,806,346个,占模型的100%(见表5)。在本文实验中,全量微调的峰值显存达到约9070MB(表9),远高于多数参数高效方法,在消费级显卡上常常逼近显存上限。

**第二,每任务存储成本高。** 全量微调为每个下游任务都产生一份完整的模型副本。ViT-Base约86M参数,以32位浮点保存约需343MB(表9实测每任务存储约343.33MB)。当部署场景涉及成百上千个任务或租户时,存储与分发成本将呈线性爆炸——10个任务即逾3GB,100个任务即逾30GB,这对边缘部署、多任务服务与模型版本管理都是沉重负担。

**第三,小数据上易过拟合并损伤预训练特征。** 下游任务往往只有数千甚至数百张标注样本。用如此小的数据集去更新近亿参数,极易过拟合;更隐蔽的风险是,激进的全局更新可能"训坏"预训练阶段精心习得的通用特征,引发**灾难性遗忘**(catastrophic forgetting),反而削弱迁移效果。本文的实验恰好为这一现象提供了佐证:在与ImageNet近域的细粒度任务上,全量微调的平均表现并非最优,反被多种参数高效方法超越(见本节末尾贡献列表)。

## 1.3 参数高效微调的动机与时代意义

针对上述痛点,**参数高效微调(Parameter-Efficient Fine-Tuning, PEFT)**应运而生。其核心思想可概括为一句话:**冻结预训练大模型的绝大部分参数,只引入或挑选极少量(常常不足总量的1%)可训练参数,以接近线性探测的代价逼近甚至超越全量微调的精度。** 这一思路最早在自然语言处理的大模型微调中被系统化,LoRA是其中的标志性工作[2]:它通过低秩旁路矩阵建模权重更新,在GPT等大模型上以极小的参数代价取得了与全量微调相当的性能,迅速成为大模型时代微调的事实标准。

PEFT的意义在大模型时代被进一步放大。一方面,模型规模的指数级增长使得全量微调的边际成本越来越难以承受,而PEFT将每任务的可训练参数与存储成本压缩两到三个数量级,使"一个冻结的共享骨干 + 若干轻量任务模块"的多任务部署成为可能。另一方面,由于绝大部分参数被冻结,PEFT天然地保护了预训练表征,在小数据上展现出更强的抗过拟合能力。可以说,PEFT不仅是一种工程优化,更代表了一种"以最小扰动适配通用基座"的方法论转变,对视觉、语言乃至多模态大模型的高效落地都具有普遍价值。

## 1.4 PEFT方法谱系简述

经过数年发展,PEFT已演化出一个方法谱系,按"那一小撮可训练参数长什么样、插在哪里"可大致归为四类(详见表1的机制汇总)。

**(一)参数选择类。** 不引入任何新结构,而是从原模型中挑选出一个极小的参数子集进行训练。代表方法是**BitFit**[3]:它冻结所有权重矩阵,仅训练网络各层的偏置项(bias)以及分类头。其直觉是,偏置如同每层的"微调旋钮",仅拧动旋钮即可让模型整体偏向新任务。该方法实现极简、参数极少(本文ViT-B上仅110,602个,占0.129%),是理想的极简对照组。

**(二)结构插入类。** 在Transformer块内部插入轻量的可训练模块。**Adapter**(Houlsby等)[7]最早在每个块中串联一个"降维-非线性-升维"的瓶颈网络;**AdaptFormer**[4]将其改造为专为ViT设计的并联瓶颈结构,挂接在MLP分支旁,以"小补习班"的形式学习任务特定的额外变换;**LoRA**[2]则另辟蹊径,认为权重更新量ΔW本质上是低秩的,因而用两个瘦长矩阵的乘积来近似它,公式为

h = W₀x + (α/r)·B·A·x

其中W₀为冻结的预训练权重,B为d×r、A为r×k的低秩矩阵,秩r远小于维度d,α为缩放系数;LoRA通常只作用于注意力的Q、V投影,且推理时可将B·A合并回W₀,实现**零额外推理延迟**。

**(三)特征调制类。** 不改动权重,而是对中间特征做轻量的逐通道线性变换。代表方法是**SSF**(Scale and Shift Features)[5]:它在若干算子之后对特征施加可学习的逐通道缩放γ与平移β,公式为

y = γ⊙x + β

其中⊙为逐元素相乘。由于该变换是线性的,推理时同样可被"吸收"进相邻层权重(重参数化),实现零额外开销。其直觉是预训练特征已足够好,只需对每个通道做"调亮度、调对比度"式的精细修正。

**(四)提示调制类。** 完全不改动骨干内部,而是在输入层做文章。**VPT**(Visual Prompt Tuning)[6]在patch token序列前拼接若干可学习的"提示token",冻结骨干仅训练这些提示,如同给模型喂入几个"暗号"以引导其走向新任务。该方法参数量最小(本文ViT-B上仅23,050个,占0.027%)。

## 1.5 当前研究中的若干问题

尽管PEFT方法层出不穷,本文在系统梳理后认为,现有研究仍存在三方面不足。**其一,缺乏统一公平的横向对比。** 不同方法源自不同论文,所用骨干、数据集、训练超参与评测协议各异,论文间报告的数字难以直接比较,研究者难以判断方法间的真实优劣。**其二,忽视了效率的多维性。** 许多工作仅以"可训练参数量"作为唯一的效率指标,却忽略了峰值显存、每任务存储、训练吞吐等同样关键的工程维度——这些维度之间往往并不一致(如某些方法参数极少却显存偏高),单一指标极易给出片面结论。**其三,缺乏组合创新与跨规模、跨域的系统考察。** 多数方法被孤立地提出与评测,鲜有研究探索不同机制(如低秩适配与特征调制)的互补性组合;同时,方法在不同骨干规模、不同域差距数据集上的行为规律也缺乏成体系的实证刻画,导致实践中"该选哪种方法"往往只能凭经验。

正是为了系统性地回应上述问题,本文构建了一个统一、公平的评测框架,在严格一致的实验协议下对8种微调策略进行多维度、跨规模、跨数据集的系统研究,并在此基础上提出一种简洁有效的组合改进方法。

在本节最后,本文主要贡献总结如下:

1) **统一的系统化评测框架。** 本文在完全一致的骨干、数据增强、优化器与评测协议下,系统对比了8种微调策略(全量微调、线性探测、BitFit、LoRA、AdaptFormer、SSF、VPT及本文提出的LoRA+SSF),覆盖8个下游数据集(CIFAR-100、Flowers-102、Pets、DTD、CIFAR-10五个通用物体/细粒度近域任务,及SVHN、EuroSAT、GTSRB三个数字/卫星/交通标志的远域任务)与主用的 ViT-S、ViT-B 两种骨干规模(ViT-L 作为后续扩展),并从精度、可训练参数、显存、存储等多个维度综合刻画,形成了一份可直接指导方法选型的完整图谱。

2) **跨域选择规律的实证发现。** 本文发现**域差距(domain gap)是决定方法选择的关键因素**。在远离ImageNet预训练分布的任务上,冻结特征不足以支撑任务,线性探测在SVHN上崩溃至仅65.08%,而依赖全局更新的全量微调反而占优(SVHN达97.34%、GTSRB达97.43%);相反,在与预训练近域的细粒度任务(Flowers、Pets、DTD)上,参数高效方法凭借对预训练特征的保护普遍优于全量微调。这一规律为"何时用PEFT、何时用全量微调"提供了清晰的判据。

3) **简洁有效的组合改进LoRA+SSF。** 本文提出将低秩权重适配(LoRA)与特征调制(SSF)相结合的LoRA+SSF方法,使两种互补机制——分别作用于权重更新与特征分布——协同发挥作用。该方法仅以0.437%的可训练参数取得8数据集平均95.26%的Top-1准确率,超越全量微调(94.93%),在全部8种方法中位列第2,仅次于参数量约其3倍的AdaptFormer(95.54%),展现出优异的精度-效率折中。

4) **骨干缩放规律的刻画。** 本文沿ViT-S→ViT-B规模轴系统考察了各方法的缩放行为,发现参数高效方法相对全量微调的优势随骨干增大而趋于稳定乃至扩大(如AdaptFormer由ViT-S的91.23%升至ViT-B的93.00%,持续领先全量微调),揭示了PEFT在更大模型上同样可靠、甚至更具吸引力的缩放性质。

5) **基于可解释性分析的机理揭示。** 本文借助CKA层级特征相似度[18]、一维损失地形[19]与逐层适配强度分析,从表征层面解释了PEFT为何往往更优:参数高效方法在浅层与中层几乎完整地保留了预训练特征,仅对高层表征施加任务特定的修正,从而既避免了灾难性遗忘,又获得了更平坦、更易优化的损失地形;而全量微调对浅层特征的过度扰动正是其在小数据近域任务上失分的根源。
