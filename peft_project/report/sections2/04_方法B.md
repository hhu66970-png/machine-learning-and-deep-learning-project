## 3.5 AdaptFormer：MLP 分支的并联瓶颈适配器

如果说 LoRA 是在注意力的"信息混合"环节做低秩修正，那么 AdaptFormer 则把适配的焦点转向 Transformer block 的另一条主干——MLP 分支，通过在其旁**并联**一个瓶颈结构的小网络来注入下游知识[4]。记 block 中第二个 LayerNorm 的输出为 h = LN₂(z′)，AdaptFormer 将原本的 MLP 子层改写为

x_out = z′ + MLP(h) + s · W_up · GELU(W_down · h)

即适配器与原始 MLP **共享同一个 LayerNorm 输出 h、并行计算后再与残差相加**。其中 W_down ∈ ℝ^{64×768} 将 768 维特征压缩到瓶颈维度 64，经 GELU 非线性激活后由 W_up ∈ ℝ^{768×64} 升回 768 维,最后乘以一个可学习的标量缩放 s 控制旁路对主通路的整体贡献强度。这一"降维—非线性—升维"的瓶颈设计,使适配器能以极少的参数学习到一个非线性的特征修正函数,其表达能力强于纯线性的 LoRA 与 SSF。

本实现的关键细节与源码逐一对应(见 src/models/peft.py 中 Adapter 类):

- **瓶颈维度**取 64,down 层(768→64,带偏置)、up 层(64→768,带偏置)。
- **零初始化保证恒等起点**:与 LoRA 同理,**up 层的权重与偏置均全零初始化**,而 down 层采用 Kaiming 均匀初始化(a=√5)。如此一来,训练起点处旁路输出 s·W_up·GELU(W_down·h) ≡ 0,整个模型严格等价于预训练模型,避免随机扰动破坏已学好的表征,训练初期稳定。
- **可学习标量 s 初始化为 0.1**:不同于将 s 固定为常数,本实现把 s 设为可训练参数并以较小的 0.1 起步,让优化器自行决定每一层适配旁路应被放大到何种强度——这相当于一个逐块的"自适应门控",在第 9 章(9.2节)逐层适配强度分析中可观察到不同深度的 s 收敛到不同量级。

**与串联 Adapter 的对比。** 经典的 Houlsby Adapter[7] 采用**串联**结构:把瓶颈模块直接插在注意力子层和 MLP 子层之后,适配信号串接进主通路(形如 h ← h + Adapter(h),且 Adapter 处于残差路径之内),因而会直接改变流向后续层的特征。AdaptFormer 的并联设计有两点本质差异:其一,原始特征流 MLP(h) 原封不动地通过,适配器只是作为一条独立旁路把修正量叠加到残差上,**预训练的前向通路被完整保留**,更契合"尽量不破坏预训练表征"的 PEFT 原则;其二,串联 Adapter 必须串在每个子层(注意力与 MLP)之后,通常需要两个适配器,而 AdaptFormer 只在 MLP 分支并联一个,位置更精简。实证上(见第 5 章),AdaptFormer 取得了全部 8 个数据集 95.54% 的最高平均精度,验证了并联瓶颈在 ViT 上的有效性。

代价方面,AdaptFormer 的参数量在本文 PEFT 方法中**最多**:每块约 (768×64+64)+(64×768+768)+1 ≈ 99,137 个,12 块叠加分类头后,ViT-B 上可训练参数共 1,197,334 个,占比 **1.376%**。此外,由于瓶颈中的 GELU 非线性,旁路无法像 LoRA/SSF 那样合并回原权重,**推理时存在少量额外计算与显存**——这是其换取更强表达能力所付出的代价。

## 3.6 SSF：特征的逐通道缩放与平移

SSF(Scale & Shift Features)持有一种与前述方法不同的适配哲学:它假设预训练特征本身已经足够好,下游适配并不需要重塑特征的"方向"或引入新的非线性,而**只需对特征分布做一次"逐通道的线性校准"**[5]:

y = γ ⊙ x + β ,  γ, β ∈ ℝ^d

其中 ⊙ 为逐元素乘,γ 为逐通道缩放向量(调整每个特征通道的幅度/对比度),β 为逐通道平移向量(调整每个通道的偏置/亮度)。直觉上,预训练域(ImageNet-21k)与下游域之间的差异,很大程度上体现为各特征通道激活统计量(均值、方差)的失配;SSF 用一对仿射参数把这种分布失配校准回来,即可让冻结的骨干特征适配新任务。

本实现的一个突出特点是**密集插入**:在每个 block 内插入 **4 处** SSF 调制(见源码 inject_ssf 与 _ssf_block_forward),分别位于——① norm1 之后(调制注意力输入)、② 注意力输出之后、③ norm2 之后(调制 MLP 输入)、④ MLP 输出之后,对应的 block 前向改写为

z′ = z + SSF_attn(MSA(SSF_n1(LN₁(z))))

z″ = z′ + SSF_mlp(MLP(SSF_n2(LN₂(z′))))

如此覆盖了 block 内注意力与 MLP 两条支路的输入与输出共四个关键节点,使逐通道校准贯穿整个前向流。**初始化**取 γ ~ N(1, 0.02²)、β ~ N(0, 0.02²),即以近似恒等映射为起点(γ≈1、β≈0,y≈x),保证温和稳定的训练起点。每块参数 4×2×768 = 6,144 个,12 块叠加分类头后 ViT-B 上共 81,418 个,占比仅 **0.095%**——是本文除 Linear Probing 与 VPT 外参数最省的方法之一。

SSF 最优雅之处在于**可重参数化、零推理开销**。由于 γ、β 是线性变换,且每处 SSF 都紧跟在一个线性运算(LayerNorm 仿射,或线性层输出 x = Wz + b₀)之后,推理时可将其代数地"吸收"进相邻权重:

γ ⊙ (Wz + b₀) + β = (diag(γ)·W)·z + (γ ⊙ b₀ + β)

吸收后网络结构、参数量、计算图与原始模型完全一致,**推理时不存在任何额外延迟或显存**。这一性质使 SSF 在部署侧极具吸引力:训练期付出极小代价,推理期则等同于一个普通的微调模型。需要指出的是,SSF 的表达力天然受限——diag(γ) 是对角变换,无法表达跨通道的方向性混合,这一局限正是 3.8 节将 LoRA 与 SSF 组合的动机之一。

## 3.7 VPT：可学习的视觉提示

VPT(Visual Prompt Tuning)采取了一条与前述"改内部参数"截然不同的路线:它**完全不改动骨干内部的任何权重**,而是在输入 token 序列中拼接若干可学习的"提示向量"(prompt token),冻结整个骨干,仅训练这些 prompt 与分类头[6]。本实现取 N = 20 个提示,提示矩阵 P ∈ ℝ^{20×768} 以截断正态分布(std=0.02)初始化,并插入在 **cls token 之后、patch token 之前**(见源码 _make_vpt_forward_features):

Z₀ = [x_cls ; P ; x_patch]

序列长度由 197 增至 217。冻结的骨干在每一层自注意力中,让 20 个 prompt token 与 196 个图像 token 及 cls token 充分交互——prompt 既能"读取"图像信息,又能把学到的任务先验"广播"回 cls 与图像 token,从而在不改动任何骨干权重的前提下,把整个模型的注意力分布"提示"向新任务偏移。其直觉类似于 NLP 中的连续提示(soft prompt):用几个可学习的"暗号 token"引导一个冻结的大模型走向特定任务。

本实现为**浅层 VPT(VPT-Shallow)**:可学习提示仅在输入层这一处注入,随后随序列一路前传,而非在每一层都注入独立的提示(VPT-Deep)。因此提示参数仅 20×768 = 15,360 个,叠加分类头后 ViT-B 上共 23,050 个,占比 **0.027%**,是除 Linear Probing 外参数最少的方法。VPT 的代价集中在推理侧:序列加长约 10%(197→217),自注意力计算量随序列长度近似二次增长而相应增加,且这一开销无法通过合并消除——这与 SSF/LoRA 的"零推理开销"形成对照。第 5 章结果亦将显示,浅层 VPT 在近域细粒度任务上表现尚可,但在远域任务(如 SVHN、GTSRB,精度仅 90.33% 与 88.78%)上明显落后,提示其"仅靠输入提示、不触碰骨干内部"的适配能力存在上限。

## 3.8 本文改进：LoRA+SSF 组合微调

在系统对比了上述各方法之后,本文提出一个简单而有效的组合改进——**LoRA+SSF**,即在同一骨干上同时注入 LoRA 与 SSF 两种模块。其设计并非两种方法的随意堆叠,而是基于二者在**适配机制上的互补性**这一明确洞察。

**动机:一改子空间,一校准分布。** 如表所示,LoRA 与 SSF 在"作用对象"和"作用方式"两个维度上恰好正交互补:

| 维度 | LoRA | SSF |
|---|---|---|
| 作用位置 | 注意力 Q/V 投影内部(权重空间) | 子层输入输出处(激活空间) |
| 作用方式 | 方向性的低秩权重更新 ΔW=BA | 逐通道的统计调制 γ⊙x+β |
| 改变了什么 | 特征"如何被混合"(子空间重定向) | 特征"如何分布"(均值/幅度校准) |
| 能力局限 | 不触碰通道统计、容量受秩 r 限制 | 对角变换、无法跨通道方向性更新 |

具体而言,LoRA 通过低秩旁路 (α/r)·B·A·x 对注意力投影施加**方向性的子空间更新**,改变的是 token 之间信息聚合的方式,但它并不显式校准各通道的激活统计;反过来,SSF 通过 γ⊙x+β 精确地**校准预训练域与下游域之间的特征分布失配**,但其对角结构(diag(γ))从原理上无法表达跨通道的方向性混合。二者**注入位置不重叠**(一个在 MSA 投影矩阵内部,一个在子层输入输出处),**机制不冗余**(一个动子空间,一个动分布),因此组合后有望同时覆盖"子空间重定向"与"分布校准"两类适配需求,取长补短。这正是单一方法各自的短板:LoRA 缺乏分布校准能力,SSF 缺乏方向性容量。

**实现。** 在同一 ViT 骨干上依次执行 LoRA 注入(r=8、α=64,作用于 Q/V 投影,同 3.4 节)与 SSF 注入(每块 4 处,同 3.6 节),解冻模式取两者可训练参数前缀的并集——"lora_" 与 "ssf_"——再加分类头(见源码 _UNFREEZE_PATTERNS["lora_ssf"] = ["lora_", "ssf_"])。一个值得强调的工程优势是:两种模块都满足"零/近恒等初始化"性质(LoRA 的 B 全零、SSF 的 γ≈1 与 β≈0),因而**组合后的训练起点仍严格接近预训练模型**,无需任何额外的稳定化技巧即可平稳训练。ViT-B 上骨干侧可训练参数为 LoRA 的 302,602 与 SSF 的 81,418 的并集(共享同一分类头),实测可训练参数共 376,330 个,占比仅 **0.437%**——不足全量的 1/200,仅比纯 LoRA(0.351%)增加约 0.086 个百分点。更难得的是,由于 LoRA 旁路可合并、SSF 仿射可吸收,**组合方法在推理时同样零额外延迟**,兼得二者的部署优势。

**预期收益与实证。** 我们预期 LoRA+SSF 能在不显著增加参数的前提下,以"子空间+分布"双重适配维持甚至小幅改善单一 LoRA 的精度,尤其在域差距较大、需要分布校准的任务上更稳。第 5 章结果与这一判断基本一致:LoRA+SSF 在 8 数据集上取得 **95.26%** 的平均精度,与纯 LoRA 的 95.25% 几乎持平(仅高 0.01 个百分点,处于多种子噪声 ±0.275 之内,不构成显著差异),并高于 SSF 的 94.84% 与 Full FT 的 94.93%,以 0.437% 的极小参数代价位列全部方法第二(仅次于参数量为其 3 倍多的 AdaptFormer 的 95.54%)。这支持了"正交机制叠加可在不增参数预算下维持综合表现"的设计思路——其价值并不在于刷新单点最高分,而在于稳健性与效率的平衡:平均上与 LoRA 持平,在低数据比例与部分细粒度近域任务上表现更稳,本文不据此对二者作强排序主张。

## 3.9 方法对比汇总

至此,本文涉及的 8 种微调策略已逐一阐明。表1 从可训练位置、核心公式/机制、推理额外开销与可训练参数占比四个维度对全部方法做统一汇总,参数占比为 ViT-B 上含分类头的实测值(精确参数个数见第 5 章表5)。

**表1 各微调方法机制汇总(ViT-B)**

| 方法 | 可训练位置 | 核心公式 / 机制 | 推理额外开销 | 可训练参数占比 |
|---|---|---|---|---|
| Linear Probing | 仅分类头 | y = W_head·z | 无 | 0.009% |
| VPT[6] | 输入提示 token + 头 | Z₀ = [x_cls; P; x_patch] | 序列加长 20 token | 0.027% |
| SSF[5] | 每块 4 处 γ、β + 头 | y = γ ⊙ x + β | 无(可吸收重参数化) | 0.095% |
| BitFit[3] | 全部偏置 b + 头 | y = Wx + b,只训 b | 无 | 0.129% |
| LoRA[2] | Q/V 投影低秩旁路 + 头 | h = W₀x + (α/r)·B·A·x | 无(可合并) | 0.351% |
| LoRA+SSF(本文) | Q/V 低秩旁路 + 4 处 γ、β + 头 | 上两行机制叠加(子空间+分布) | 无(均可合并/吸收) | 0.437% |
| AdaptFormer[4] | MLP 并联瓶颈 + 头 | x + MLP(h) + s·Up(GELU(Down(h))) | 少量旁路计算 | 1.376% |
| Full FT | 全部参数 | min L(D; θ) | 无 | 100.000% |

横向审视此表可提炼出三条结构性规律。其一,**推理开销与训练机制解耦**:除 AdaptFormer(GELU 非线性不可合并)与 VPT(序列加长不可消除)外,其余方法均可在推理时还原为标准 ViT,做到零额外开销——这意味着部署成本主要取决于"是否引入不可合并的非线性/序列变动",而非可训练参数的多少。其二,**参数占比横跨四个数量级**:从 Linear Probing 的 0.009%、VPT 的 0.027%,到 SSF/BitFit 的 0.1% 量级,再到 LoRA 系的 0.35%~0.44% 与 AdaptFormer 的 1.376%,最后跃升至 Full FT 的 100%。其三,即便参数最多的 AdaptFormer,与全量微调之间仍隔着近两个数量级,而本文 LoRA+SSF 在引入两种互补机制后,占比仍严格控制在 0.5% 以内。

图1 以对数横轴(可训练参数量)对 8 个数据集分别绘制各方法的 Top-1 准确率,直观呈现了"精度—参数"关系。

![图1 准确率随可训练参数量的变化(8 数据集)](ef_acc_vs_params.png)

图中最值得注意的现象是:在跨越四个数量级的参数谱系上,**准确率并非随可训练参数单调上升**。Full FT 以 100% 的参数占比并未在多数数据集上取得最优,反而被参数量仅为其 1/200~1/70 的 LoRA、LoRA+SSF、AdaptFormer 在平均精度上反超;而 SSF、BitFit 以 0.1% 量级的参数即逼近 Full FT。换言之,微调收益的决定因素并非"训练了多少参数",而是"**参数插在哪、以何种机制起作用**"——LoRA 的方向性子空间更新与 SSF 的分布校准之所以高效,正因其精准命中了下游适配真正需要改动的自由度,而非盲目放开全部参数。这一"少即是多"的非单调关系,为后续第 4 章对各方法适用边界(近域/远域、数据规模、骨干尺度)的深入剖析奠定了基调,也从根本上回答了本文的核心追问:参数高效微调不是全量微调的廉价替代,而是一种在归纳偏置上更契合迁移学习本质的适配范式。
