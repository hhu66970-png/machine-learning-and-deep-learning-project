# 3. 研究方法

本章给出本文研究所依赖的统一形式化框架与各微调策略的精确定义。本文的写作原则是:任何对方法的文字描述都必须与实际运行的源码实现(src/models/peft.py)严格一致——不仅给出"教科书式"的理想公式,更要点明本文实现中真实采用的注入位置、初始化方案、缩放系数与参数计数口径,因为这些"实现细节"往往才是决定一个参数高效微调(PEFT)方法实测表现的关键。本节(3.1–3.4)先确立统一记号、回顾视觉Transformer(ViT)的结构骨架与"插补丁"位置,随后依参数量从大到小、从基线到具体方法的顺序,依次刻画Full Fine-Tuning与Linear Probing两条基准线、BitFit与LoRA两种代表性方法;AdaptFormer、SSF、VPT以及本文提出的组合改进LoRA+SSF将在本章后半部分(3.5–3.8)给出。

## 3.1 问题定义与统一框架

### 形式化:从全量微调到受限子空间优化

考虑一个在大规模上游数据(本文为ImageNet-21k,约1400万张图、约2万类[14])上预训练得到的视觉Transformer,其全部参数记为 θ₀ ∈ ℝᴺ。这里 N 是骨干网络的参数总数,对ViT-B/16而言 N ≈ 86M。给定一个下游任务及其训练集 D = {(xᵢ, yᵢ)},微调的目标是寻找一组参数 θ,使经验风险最小:

min_θ  L(θ) = (1/|D|) Σᵢ ℓ(f(xᵢ; θ), yᵢ)

其中 f(·; θ) 为模型前向映射,ℓ 为交叉熵损失。**全量微调(Full Fine-Tuning)**直接把全部 N 个分量都当作自由变量,即在整个 ℝᴺ 空间内搜索 θ,这等价于学习一个与骨干同维的更新量 Δθ = θ − θ₀ ∈ ℝᴺ。当下游数据较少而 N 极大时,这种"自由度过剩"既带来显存与存储负担,也容易把预训练学到的优质特征"训坏",诱发过拟合与灾难性遗忘。

PEFT的核心洞见可以用一句话刻画:**把更新量 Δθ 约束在一个维度极低的子空间内**。形式上,引入一组维度远小于 N 的可训练参数 φ ∈ ℝᴹ(M ≪ N),并规定 θ = θ₀ + g(φ),其中 g 是一个把低维 φ 映射回 N 维参数空间(或等价地映射为对前向计算的扰动)的结构化算子。于是优化问题被改写为:

min_φ  L(θ₀ + g(φ)),   M ≪ N

冻结的骨干 θ₀ 不再参与梯度更新。不同PEFT方法的全部差异,实质上只在于"算子 g 长什么样、把这 M 个自由度放在网络的哪个位置":BitFit让 φ 仅取偏置维度;LoRA让 g(φ) 是若干权重矩阵的低秩增量;SSF让 g 作用为逐通道仿射;VPT甚至不修改任何已有权重,而是在输入序列上追加可学习的token。理解了"Δθ被限制在极低维子空间"这一共性,后续各方法便可在同一框架下被统一解读与比较。这一受限优化视角也解释了PEFT为何在近域小数据上常常优于全量微调:更小的有效假设空间带来更强的隐式正则,降低了在有限样本上过拟合的风险。

### ViT block结构与"插补丁"位置

要精确说明各方法把可训练参数插在何处,需先回顾ViT[1]的基本计算单元。一张 224×224 的图像被切成 16×16 的patch并线性投影为patch嵌入,拼接一个可学习的分类token([CLS]),加上位置编码后,送入 L 个堆叠的Transformer block(ViT-B为 L=12,嵌入维度 d=768)。每个block采用Pre-LN结构,由"多头自注意力(MSA)子层"与"前馈网络(MLP)子层"组成,各自带一条残差连接:

x' = x + DropPath(LS₁(MSA(LN₁(x))))
x'' = x' + DropPath(LS₂(MLP(LN₂(x'))))

其中 LN 为LayerNorm,LS(LayerScale)为可选的逐通道缩放,DropPath为随机深度。MSA子层内部先由一个**融合的qkv线性层**把输入投影为查询、键、值三者拼接的张量(在timm实现中,qkv是一个 d→3d 的单一 nn.Linear,输出按通道切分为 q、k、v),经多头注意力聚合后由输出投影 proj 还原维度。MLP子层则是"Linear(d→4d) → GELU → Linear(4d→d)"的瓶颈放大结构。这一结构暴露出若干天然的"插补丁"位置,本文各方法分别利用了它们:LoRA介入融合qkv投影内部、给q与v加低秩旁路;AdaptFormer在MLP子层旁并联瓶颈适配器;SSF在LN之后、注意力与MLP输出之后插入逐通道仿射;VPT在 [CLS] 之后、patch序列之前插入提示token。无论采用哪种方法,本文都遵循统一约定:**分类头(classifier head)始终保持可训练**——因为下游任务的类别空间与上游完全不同(例如从2万类切换到102类花),线性映射到新类别集合的权重无法从预训练继承,必须从头学习。在源码 apply_peft 中,这一约定通过在冻结骨干后显式对 model.get_classifier().parameters() 调用 requires_grad_(True) 来落实,对除Full FT外的所有方法一致生效。

### 统一记号

为后文统一,约定如下记号。设嵌入维度为 d(ViT-S/B/L 分别为 384/768/1024),block数为 L(分别为 12/12/24)。对单个权重矩阵,记其输入维度 d_in、输出维度 d_out。可训练参数总数记为 M,其占骨干总参数 N 的比例记为 M/N(本文表5列出各方法在ViT-B上的实测值)。一个block的前向中,记注意力子层的归一化输出为 h_attn = LN₁(x)、MLP子层的归一化输出为 h_mlp = LN₂(x')。各PEFT模块的输出统一以"在原计算路径上叠加一个低维扰动项"的形式表达,便于在3.2–3.8中横向对照。

## 3.2 两条基准线:Full Fine-Tuning 与 Linear Probing

任何PEFT方法的价值都需放在两条基准线之间衡量:全量微调给出"精度上限"参照,线性探测给出"代价下限"参照。本文以这两条线作为标尺,PEFT的理想目标是"以接近Linear Probing的代价,逼近甚至超过Full FT的精度"。

**Full Fine-Tuning(全量微调,精度上限)。**该策略不冻结任何参数,可训练集合即 φ = θ,自由度 M = N。在源码中对应 apply_peft 的 full_ft 分支:对 model.parameters() 全体调用 requires_grad_(True),不注入任何额外模块。其优势是假设空间最大、表达力最强,在与上游差异极大的远域任务上往往能取得最佳精度——本文实验中,在SVHN(97.34%)与GTSRB(97.43%)这类街景数字与交通标志数据上,Full FT在ViT-B上即为或接近最强(见表4),印证了"远域任务需要骨干深度重塑"的判断。但其代价同样最高:其一,需要为全部86M参数计算并存储梯度,且AdamW[12]优化器需为每个参数额外维护一阶动量与二阶动量两份状态,使训练显存压力显著上升——本文实测Full FT峰值显存约9070MB(表9),处于最高一档;其二,每个下游任务都要保存一份完整的骨干副本,本文实测每任务存储约343.33MB(表9),是Linear Probing(0.14MB)的约2450倍,在多任务部署场景下成本难以接受;其三,在近域小数据上,过大的自由度反而损害泛化——本文中Full FT在CIFAR-100(92.41%)、Pets(93.76%)、Flowers-102(99.43%)等近域任务上均不及多数PEFT方法,其8数据集平均(94.93%)被AdaptFormer、本文LoRA+SSF与LoRA三种方法超越(见表4与第5章(5.2节)关键发现)。这恰恰说明"上限"并非处处最优:全量微调的优越性高度依赖任务与上游的域差距。

**Linear Probing(线性探测,代价下限)。**该策略把整个骨干完全冻结,只训练新接入的线性分类头,即 φ 仅含分类头参数。在源码中对应 linear_probe 分支:其解冻模式 _UNFREEZE_PATTERNS["linear_probe"] 为空列表 [],不匹配任何骨干参数,仅由统一的"分类头始终可训练"约定保留分类头梯度。其优点是代价极低、绝不破坏预训练特征:本文实测可训练参数仅7,690个(占比0.009%),峰值显存仅1622MB、每任务存储0.14MB,均为所有方法中最低(表5、表9)。但其表达力也最弱:骨干特征完全不为新任务适配,精度在域差距大的任务上急剧下滑。本文中Linear Probing的8数据集平均仅87.06%,且在SVHN上低至65.08%(表4)——这是全表的最差单元格,直接量化了"冻结特征不足以表征远域数字图像"这一结论。Linear Probing因而成为评判PEFT是否"物有所值"的下界:任何方法若不能以可接受的额外代价显著超过它,就失去了存在意义。

## 3.3 BitFit:只训练偏置项

BitFit[3]是参数最省的代表性方法之一,其思想极为简洁:**冻结所有权重矩阵 W,只训练网络中的偏置项 b(外加分类头)**。对任一仿射层,前向为

y = Wx + b

BitFit保持 W 不动,仅让 b 可学习。其直觉是把偏置视作每一层的"微调旋钮":在权重所定义的特征几何不变的前提下,仅平移各层激活的中心,即可让冻结骨干的响应整体偏向新任务。在源码中对应 bitfit 分支,其解冻模式为 [".bias"]——遍历 model.named_parameters(),凡参数名包含子串 ".bias" 者即解冻。这一字符串匹配会覆盖ViT骨干中所有带偏置的模块:各LayerNorm的 bias、qkv投影与输出投影proj的 bias、MLP两个线性层的 bias 等,同时分类头再由统一约定解冻。本文实测BitFit在ViT-B上可训练参数为110,602个,占比仅0.129%(表5)。

尽管参数极少,BitFit的精度并不弱:其8数据集平均达94.63%(表4),已逼近Full FT(94.93%),并显著超过Linear Probing(87.06%)。值得注意的是,BitFit在近域细粒度任务上表现稳健(Flowers-102 99.67%、CIFAR-100 93.54%),但在远域任务上短板明显——SVHN仅95.02%、GTSRB仅96.01%,均落后于LoRA、AdaptFormer等带权重增量的方法。这一对比揭示了"只调偏置"的内在局限:偏置只能平移激活、无法旋转或重塑特征子空间,当下游域要求的特征变换超出"平移"所能表达的范围时,BitFit便力有不逮。因此本文将BitFit定位为极简对照组:它确立了"仅靠最廉价的自由度能走多远"的参照点,而LoRA、AdaptFormer等方法相对它的增益,正可归因于"引入权重级(而非仅偏置级)适配"所带来的额外表达力。

## 3.4 LoRA:低秩适配

### 核心思想与公式

LoRA(Low-Rank Adaptation)[2]最初为大语言模型设计,现已成为PEFT的事实标准。其出发点是:全量微调对某权重矩阵 W₀ ∈ ℝ^{d_out×d_in} 学习的更新量 ΔW,虽与 W₀ 同形,但其**内在秩往往很低**——即真正"有用"的更新只分布在一个低维子空间中。据此,LoRA不直接学习稠密的 ΔW,而是用两个瘦长矩阵的乘积对其低秩近似:ΔW = B·A,其中 A ∈ ℝ^{r×d_in}、B ∈ ℝ^{d_out×r},秩 r ≪ min(d_in, d_out)。前向计算在冻结的 W₀ 之上叠加这一低秩旁路,并乘以缩放系数 α/r:

h = W₀x + (α/r)·B·A·x

参数量由稠密更新的 d_out×d_in 降为 r×(d_in + d_out)。缩放系数 α/r 的作用是解耦"秩 r"与"更新幅度":当改变 r 时,以 α/r 归一化可使旁路输出的量级大致稳定,从而减少对学习率的重新调参需求。

### 本文实现细节(与源码一致)

本文实现见 src/models/peft.py 中的 LoRAqkv 模块与 inject_lora 注入函数,有若干与"教科书公式"一致但需明确点出的工程细节:

(1)**仅注入注意力的q与v,且需拆分融合qkv。**LoRA经验上加在注意力投影即可获得主要收益。本文遵循原文设置,只对查询(q)与值(v)投影施加低秩增量,不动键(k)与输出投影proj。由于timm的ViT把q、k、v合并为一个 d→3d 的融合 nn.Linear(self.qkv),本文的做法是用 LoRAqkv 整体包裹这个冻结的融合层:前向中先计算冻结qkv的完整输出 qkv = self.qkv(x),再沿最后一维按 [0:d]、[d:2d]、[2d:3d] 切出 q、k、v 三段,仅对q段与v段各自叠加一条独立的低秩旁路,k段保持不变,最后重新拼接为 [q; k; v]。对应核心代码为

q, k, v = qkv[..., :d], qkv[..., d:2d], qkv[..., 2d:]
q = q + lora_B_q(lora_A_q(x)) · scaling
v = v + lora_B_v(lora_A_v(x)) · scaling

其中 scaling = α/r。注意q与v各拥有独立的 (A, B) 对,因此每个block实际引入两条低秩旁路。

(2)**A为Kaiming初始化、B为零初始化,保证起点等价于预训练。**本文对 lora_A_q、lora_A_v 的权重采用 Kaiming均匀初始化(nn.init.kaiming_uniform_,a=√5),对 lora_B_q、lora_B_v 的权重采用零初始化(nn.init.zeros_)。这一非对称初始化是LoRA的关键技巧:由于 B=0,旁路输出 B·A·x 在训练起点恒为零,故 h = W₀x,模型行为与未注入LoRA的预训练骨干完全等价。这避免了在训练开始时对预训练特征引入随机扰动,使优化从一个"已知良好"的起点平滑出发;同时 A 用Kaiming随机初始化打破对称、保证一旦 B 开始更新,两个矩阵能学到不同方向。所有四个低秩矩阵均设 bias=False(nn.Linear(..., bias=False)),即低秩旁路不含偏置项。

(3)**缩放 α/r 与可选dropout。**模块以 self.scaling = alpha / r 实现缩放;本文主实验取 r=8、α=64,故 scaling=α/r=8.0。前向中在低秩旁路的输入端可施加dropout(self.drop),主实验中dropout=0.0。

(4)**推理可合并、零额外延迟。**由于低秩旁路是对q、v的线性叠加,训练完成后可将 (α/r)·B·A 直接加回融合qkv权重的对应分块(q与v对应的行块),得到一个与原结构完全相同的等效qkv层。因此LoRA在推理期不引入任何额外算子或延迟,这是其相对AdaptFormer(并联分支必须在推理期执行)的一个工程优势。

### 参数量与消融

依上述实现,LoRA在ViT-B上的可训练参数量(不含分类头)可由下式估计:

M_LoRA = r × (d_in + d_out) × 2 × L

其中因子2来自"每block对q与v各一条旁路",L为block数。对ViT-B(d_in=d_out=768、L=12、r=8),代入得 8×(768+768)×2×12 = 294,912,与表5中LoRA实测可训练参数302,602个(占比0.351%,差值即分类头部分)相吻合。

LoRA的实测表现优异:8数据集平均95.25%(表4),不仅显著超过Linear Probing与BitFit,也超过Full FT(94.93%),位列全部方法第三,且在Flowers-102上以99.71%与AdaptFormer并列最高。本文还以DTD为基准对秩 r 做了系统消融(表8、图10):r 从1增至32时,可训练参数由73,007增至1,215,791,Top-1由80.48%总体趋势性上升(非严格单调,r=8 处略有回落)至82.87%,其中 r=1→2 几乎无增益(80.48%→80.53%)、r=4 出现明显跃升(81.60%)、r=16/32 达到高位(82.66%/82.87%)。这表明对DTD这类纹理任务,极低秩(r=1,2)不足以承载所需的特征适配,而 r≥4 后收益趋于饱和、边际递减——增大 r 带来的参数膨胀(r=32时参数约为 r=4时的6.6倍)并未换来等比例的精度提升。这一观察支撑了本文主实验选取 r=8 作为"精度—参数"折中点的决策,并为3.8节中LoRA+SSF组合改进的设计提供了依据:与其继续堆高LoRA的秩,不如引入一种正交的、互补的适配自由度(SSF的逐通道仿射)来进一步逼近精度上限。
